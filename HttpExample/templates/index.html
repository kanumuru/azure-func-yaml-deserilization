YAML Formatting Guide

What is YAML?

YAML (YAML Ain't Markup Language) is a human-readable data serialization format. It is commonly used for configuration files and data exchange between programming languages.

Basic YAML Syntax

YAML uses indentation and colons to define its structure. Here's an example:


key1: value1
key2:
  - item1
  - item2
key3:
  subkey1: subvalue1
  subkey2: subvalue2

Key-Value Pairs

In YAML, data is represented using key-value pairs. Keys and values can be strings, numbers, booleans, or even nested objects. Here's an example:


name: John Doe
age: 25
isStudent: true

Lists and Arrays

YAML supports lists and arrays. You can represent them using dashes followed by a space (-) for each item. Here's an example:


fruits:
  - apple
  - banana
  - orange

Nested Objects

You can nest objects within YAML. Indentation is used to indicate the nested structure. Here's an example:


person:
  name: John Doe
  age: 25
  address:
    street: 123 Main St
    city: Exampleville
    country: USA

Comments

YAML allows comments to provide additional information. Comments start with the hash symbol (#). Here's an example:


# This is a comment
name: John Doe # This is another comment

YAML Anchors and Aliases

YAML provides a way to reference values using anchors and aliases. An anchor is defined using the ampersand (&) symbol, and an alias is defined using the asterisk (*) symbol. Here's an example:


defaults: &defaults
  timeout: 30
  retries: 3

server1:
  <<: *defaults
  address: 192.168.1.100

server2:
  <<: *defaults
  address: 192.168.1.101
